# 研究生毕业论文对齐设计

日期：2026-03-10

## 目标

围绕 `references/江苏海洋大学研究生硕士学位论文撰写模版` 建立一条可持续迭代的研究生论文模板对齐链路，而不是一次性的人工调样式。

本次只专注研究生毕业论文主模板，不做额外表单。

## 约束

- 参考同级本科项目的工程标准，不走纯手工排版。
- 必须先拆解参考文档为 XML 基线，再做 LaTeX 实现。
- 必须以 `pytest` 作为一等产物，不能只靠人工肉眼判断。
- 必须以 WPS 导出 PDF 作为视觉真值。
- UX 要友好：模板默认结构清晰、入口简单、示例内容可直接替换、测试失败信息可读。

## 推荐方案

采用“本科基线迁移 + 研究生重构”方案：

1. 固化研究生参考资产。
2. 建立研究生仓库自己的 `jougraduate.cls`。
3. 沿用本科项目的对齐工程思路：
   - XML 真值
   - PDF 真值
   - 页面锚点
   - 关键页面渲染对比
   - `pytest` 合同

## 输入基线

- 原始来源：
  - `references/江苏海洋大学研究生硕士学位论文撰写模版.doc`
- OpenXML 中间件：
  - `references/江苏海洋大学研究生硕士学位论文撰写模版.docx`
- XML 真值：
  - `references/unpacked/...`
- 视觉真值：
  - `references/江苏海洋大学研究生硕士学位论文撰写模版.pdf`
- 人类可读规则摘要：
  - `references/handbook.md`

判断优先级：

1. WPS 导出 PDF 决定视觉基线
2. Word XML 决定结构基线
3. `handbook.md` 只记录稳定规则，不覆盖前两者

首轮纳入研究生论文主模板的页面族：

1. 封面
2. 学位论文使用授权声明
3. 内封
4. 论文审阅认定书
5. 致谢
6. 中文摘要
7. 英文摘要
8. 中文目录
9. 英文目录
10. 图清单
11. 表清单

14. 结论
15. 参考文献
16. 程序或附录样页

19. 学位论文数据集

明确排除：

- `Extended Abstract`
  - 参考 PDF 已明确标注“硕士学位论文不需要写”

## 实现顺序

1. 固化基线资产
   - 生成 `.docx`
   - 解包 XML
   - 提取并整理 `handbook.md`
2. 建立最小可编译主类
   - `jougraduate.cls`
   - `main.tex`
3. 分页族收敛
   - 第一轮：封面、声明、内封、审阅认定书
   - 第二轮：致谢、中英文摘要、目录、图表清单、变量表
   - 第三轮：正文、结论、参考文献、作者简历、原创性声明、数据集
4. 增加渲染对比和诊断产物
5. 补 README 和报告

## pytest 合同

### 1. 资源合同

- 参考 `.doc`、`.docx`、`unpacked XML`、WPS PDF 必须存在。

### 2. 页面顺序合同

- 模板页面顺序必须与研究生参考文档一致。

### 3. 页面锚点合同

- 关键页面必须能被稳定短语唯一定位。

### 4. PDF 合同

- 页数、A4 规格、页面朝向必须稳定。

### 5. 字体合同

- 字体模式必须可检测。
- 无标准商用字体时必须稳定 fallback。

### 6. 版式合同

- 封面、摘要、目录、正文页关键文本块的位置必须落在容忍区间内。

### 7. 内容合同

- 默认不生成 `Extended Abstract`。
- 目录默认只到二级标题。
- 中英文摘要、关键词、参考文献等关键结构必须存在。

## UX 设计要求

- `main.tex` 保持最小入口，用户只需修改元数据和正文内容。
- 内容文件按页面族拆分，命名直观。
- 示例文本保留替换提示，不保留混乱占位。
- 报错信息尽量中文化，方便学生定位问题。
- 默认编译路径和测试路径尽量短，避免额外脚本心智负担。

## 本轮不做

- 配套表单
- Overleaf 专用裁剪包
- 与研究生论文无关的本科模板迁移
- 一次性逐像素复刻全部 22 页的全部细节
