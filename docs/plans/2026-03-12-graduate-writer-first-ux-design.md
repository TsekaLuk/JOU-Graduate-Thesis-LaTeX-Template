# 研究生论文模板 Writer-First UX 收敛设计

日期：2026-03-12

## 目标

在保留当前研究生模板对齐成果的前提下，把模板进一步收敛成更像成熟高校 LaTeX 模板的写作产品：

- 用户第一次打开仓库就知道从哪开始写
- 新增章节不需要手工复制文件和维护聚合入口
- README 与使用指南优先服务“写论文的人”，不是“维护样张的人”
- 自动目录、图表清单、参考文献继续走正常 LaTeX 链路

## 不做的事

- 不把模板重新改回静态样张工程
- 不为了追求页面相似度而手写目录或图表清单
- 不扩展到本科模板的配套表单范围

## 方案选择

### 方案 A：Writer-First 收敛

保留当前锁版页与对齐基线，重点优化写作入口、章节脚手架和 onboarding 文档。

优点：

- 最像主流高校模板的使用方式
- 不破坏当前对齐与测试体系
- 对学生最友好

缺点：

- 需要继续维护“样页基线”和“真实写作入口”的边界

### 方案 B：继续收目录/清单到更像参考件

优点：

- 页面视觉更像学校 Word 样张

缺点：

- 很容易牺牲模板可写性
- UX 会倒退

本轮采用方案 A。

## 具体设计

### 1. 写作入口

保留 `contents/setup.tex` 作为集中配置入口，但进一步强化它的“第一次打开就能改”的角色：

- 用分组注释区分元数据、模式、页面开关
- 文档明确告诉用户：只改 `setup + frontmatter + chapters + backmatter`

### 2. 正文章节工作流

补一个真正的章节脚手架命令：

- 新增 `scripts/new_chapter.py`
- Makefile 增加 `make new-chapter`
- 自动从 `_template.tex` 生成新章节
- 自动接入 `contents/chapters/mainmatter.tex`

### 3. 文档 UX

README 与 `docs/guides/usage.md` 进一步收敛成：

- 3 分钟上手
- 最小写作路径
- 仓库结构图
- 新增章节的一键命令
- “哪些文件别碰”的明确边界

### 4. 合同测试

补仓库级合同，确保新增 UX 能力不会回退：

- `make new-chapter` 目标存在
- `scripts/new_chapter.py` 存在且文档可发现
- README 和 usage 指向新的章节工作流

## 成功标准

- 用户不阅读类文件也能开始写论文
- 新增章节不再需要手工复制和维护正文聚合入口
- README 与 usage 更短、更直接、更像高校模板
- `pytest` 继续全绿
